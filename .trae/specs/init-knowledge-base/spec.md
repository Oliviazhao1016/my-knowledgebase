# 个人知识库 (Personal Knowledge Base) MVP V1.0 Spec

## Why

互联网产品经理面临工作资料（文档、PRD、会议纪要）和碎片化输入（播客、推文、灵感）分散的问题，导致回查成本高且难以建立关联。本项目旨在打造一个统一的网页知识库，支持收录各种输入形式，并提供带来源追溯的问答、文档关系建立及局部图谱查看功能。

## What Changes

* 构建「个人知识库」前端交互框架，采用黑色为主色调+炫彩点缀的高端审美设计（Dark theme + Vibrant accents），利用 `frontend-design` 规范打造无“廉价AI感”的体验。

* 开发一体化输入区，支持上传文件、粘贴链接、输入文本及直接提问。

* 开发三大核心页面状态及平滑切换逻辑：

  * **首页默认态**：顶部一体化输入区、中部知识库列表、右侧常驻助手。

  * **聊天展开态**：左侧折叠知识列表、中间 Chat 对话主区、右侧信息区（来源片段、相关文档）。

  * **文档详情态**：中间原始内容、右侧信息区（局部关系图谱、来源、标签、候选关联）。

* 建立 RAG 问答与引用机制的前端展示交互（来源片段高亮、点击跳转）。

* 引入局部关系图谱（Network Graph）可视化展示文档间的关联关系。

## Impact

* Affected specs: 知识录入界面、知识检索与展示、智能问答组件、知识图谱可视化。

* Affected code: 全新的知识库前端模块（或独立项目），涉及路由状态管理、组件库及自定义炫彩主题样式架构。

## ADDED Requirements

### Requirement: 黑色+炫彩的前端美学设计

系统 SHALL 提供极具审美的高级暗黑主题，并配以高饱和度的炫彩强调色（如草绿色），结合毛玻璃（Glassmorphism）、微小动画（Micro-interactions）及流体渐变背景（Fluid Gradients）。

#### Scenario: 页面交互体验

* **WHEN** 用户滚动或悬浮核心组件（卡片、按钮、输入框）时

* **THEN** 触发平滑的 CSS 过渡动画，边框或背景浮现炫彩光晕，提升整体质感与沉浸感。

### Requirement: 状态无缝切换

系统 SHALL 提供首页、聊天页、文档详情页之间的平滑过渡切换逻辑。

#### Scenario: 从列表进入详情页并提问

* **WHEN** 用户点击知识列表中的文档，接着在右侧助手区发送消息

* **THEN** 界面平滑过渡至“聊天展开态”，左侧列表收缩为侧边栏，中间变为聊天区，右侧显示来源片段。
